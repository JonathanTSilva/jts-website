# Home Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Redesign the home page with a full-viewport two-column hero, bento project grid, horizontal-scroll publications row, expanded blog/notes sections, ContactSection removal, pill LanguageSwitcher, restructured two-row footer, and placeholder legal pages.

**Architecture:** Changes span `Hero.astro`, `HomeSections.astro`, `ProjectCard.astro`, `Footer.astro`, `LanguageSwitcher.astro`, both `index.astro` pages, `src/content/portfolio/types.ts`, and two new page stubs. The hero is broken into left/right columns within the existing component. No new third-party dependencies are introduced. All styling uses CSS custom properties from `src/styles/tokens.css` exclusively.

**Tech Stack:** Astro 5, TypeScript, vanilla JS inline scripts, CSS custom properties, Playwright E2E (behavioral assertions only, no CSS property assertions)

**Spec:** `docs/superpowers/specs/2026-03-21-home-page-design.md`

## Implementation Notes

**Token discipline:** Every CSS value in this plan uses a token from `src/styles/tokens.css`. The only permitted hardcoded values are the `#fff` in `.btn-primary` (already global) — this plan introduces no new hardcoded colors.

**BaseLayout main padding:** `BaseLayout.astro` applies `padding-block: var(--space-12)` and `max-width: var(--container-max)` to `<main>`. The hero section is a direct child of `<main>`, so its two-column grid inherits the container constraint.
